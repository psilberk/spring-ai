= Oracle Embeddings

Spring AI supports Oracle Database embeddings through `OracleEmbeddingModel`, backed by Oracle AI Vector Search and the `DBMS_VECTOR_CHAIN` package.

The model can run with in-database embeddings (provider `database`) or with external providers configured through Oracle embedding preferences.

== Prerequisites

* Oracle Database 23ai (23.4+) with AI Vector Search enabled.
* A configured JDBC `DataSource` for your Oracle instance.
* If you load ONNX models into Oracle:
** local strategy: Oracle directory + ONNX file + privileges such as `CREATE ANY DIRECTORY` and `CREATE MINING MODEL`.
** cloud strategy: an Oracle credential (optional for pre-authenticated URLs) and ONNX cloud URI.

== Add Repositories and BOM

Spring AI artifacts are published in Maven Central and Spring Snapshot repositories.
Refer to the xref:getting-started.adoc#artifact-repositories[Artifact Repositories] section to add these repositories to your build system.

To help with dependency management, Spring AI provides a BOM (bill of materials) to ensure that a consistent version of Spring AI is used throughout the entire project.
Refer to the xref:getting-started.adoc#dependency-management[Dependency Management] section to add the Spring AI BOM to your build system.

== Auto-configuration

[NOTE]
====
There has been a significant change in the Spring AI auto-configuration, starter modules' artifact names.
Please refer to the https://docs.spring.io/spring-ai/reference/upgrade-notes.html[upgrade notes] for more information.
====

Spring AI provides Spring Boot auto-configuration for the Oracle Embedding Model.
To enable it add the following dependency to your project's Maven `pom.xml` file:

[source,xml]
----
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-oracle</artifactId>
</dependency>
----

or to your Gradle `build.gradle` build file.

[source,groovy]
----
dependencies {
    implementation 'org.springframework.ai:spring-ai-starter-model-oracle'
}
----

TIP: Refer to the xref:getting-started.adoc#dependency-management[Dependency Management] section to add the Spring AI BOM to your build file.

The Oracle embedding model requires a Spring `DataSource`.
For example:

[source,yaml]
----
spring:
  datasource:
    url: jdbc:oracle:thin:@//localhost:1521/freepdb1
    username: mlops
    password: mlops
  ai:
    model:
      embedding: oracle
    oracle:
      embedding:
        options:
          model: ALL_MINILM_L12_V2
        preferences:
          provider: database
          model: ALL_MINILM_L12_V2
----

=== Embedding Properties

[NOTE]
====
Enabling and disabling of the embedding auto-configurations are now configured via top-level properties with the prefix `spring.ai.model.embedding`.

To enable, set `spring.ai.model.embedding=oracle` (it is enabled by default).

To disable, set `spring.ai.model.embedding=none` (or any value which doesn't match `oracle`).
====

==== Retry Properties

The prefix `spring.ai.retry` is used as the property prefix that lets you configure the retry mechanism for the Oracle Embedding model.

[cols="3,5,1", stripes=even]
|====
| Property | Description | Default

| spring.ai.retry.max-attempts   | Maximum number of retry attempts. |  10
| spring.ai.retry.backoff.initial-interval | Initial sleep duration for the exponential backoff policy. |  2 sec.
| spring.ai.retry.backoff.multiplier | Backoff interval multiplier. |  5
| spring.ai.retry.backoff.max-interval | Maximum backoff duration. |  3 min.
| spring.ai.retry.on-client-errors | If false, throw a `NonTransientAiException`, and do not attempt retry for client-style failures. | false
| spring.ai.retry.exclude-on-http-codes | List of HTTP status codes that should not trigger a retry. | empty
| spring.ai.retry.on-http-codes | List of HTTP status codes that should trigger a retry. | empty
|====

==== Configuration Properties

[cols="3,5,1", stripes=even]
|====
| Property | Description | Default

| spring.ai.model.embedding | Enables Oracle embedding model selection. | oracle
| spring.ai.oracle.embedding.initialize-on-startup | Load ONNX model when the embedding bean initializes. | false
| spring.ai.oracle.embedding.onnx-directory-alias | Oracle directory alias for local ONNX loading. | -
| spring.ai.oracle.embedding.onnx-file | Local ONNX filename used with `onnx-directory-alias`. | -
| spring.ai.oracle.embedding.onnx-model-name | Oracle model name used during ONNX load. | -
| spring.ai.oracle.embedding.onnx-credential | Oracle credential for cloud ONNX loading. | -
| spring.ai.oracle.embedding.onnx-uri | Cloud URI for ONNX loading. | -
| spring.ai.oracle.embedding.options.model | Model identifier used in Oracle embedding calls. | database
| spring.ai.oracle.embedding.options.dimensions | Optional embedding dimensions. | -
| spring.ai.oracle.embedding.options.proxy | Optional proxy passed to `utl_http.set_proxy`. | -
| spring.ai.oracle.embedding.options.batching | Whether batched calls use `utl_to_embeddings`. | true
| spring.ai.oracle.embedding.options.metadata-mode | `Document` metadata extraction mode. | EMBED
| spring.ai.oracle.embedding.preferences.provider | Oracle embedding provider (for example `database`, `ocigenai`). | database
| spring.ai.oracle.embedding.preferences.model | Provider model name. | database
| spring.ai.oracle.embedding.preferences.credential-name | Oracle credential name for provider access. | -
| spring.ai.oracle.embedding.preferences.url | Provider endpoint URL. | -
| spring.ai.oracle.embedding.preferences.transfer-timeout | Provider transfer timeout. | -
| spring.ai.oracle.embedding.preferences.max-count | Max input count. | -
| spring.ai.oracle.embedding.preferences.batch-size | Provider batch size. | -
|====

[IMPORTANT]
====
When `initialize-on-startup=true`, configure only one ONNX load strategy:

* local: `onnx-directory-alias` + `onnx-file`
* cloud: `onnx-credential` + `onnx-uri`

Do not combine both in the same configuration.
====

== Runtime Options [[embedding-options]]

Use https://github.com/spring-projects/spring-ai/blob/main/models/spring-ai-oracle/src/main/java/org/springframework/ai/oracle/embedding/OracleEmbeddingOptions.java[OracleEmbeddingOptions] and https://github.com/spring-projects/spring-ai/blob/main/models/spring-ai-oracle/src/main/java/org/springframework/ai/oracle/embedding/OracleEmbeddingPreferences.java[OracleEmbeddingPreferences] to override per-request behavior.

[source,java]
----
EmbeddingResponse response = embeddingModel.call(
    new EmbeddingRequest(
        List.of("Hello World", "World is big and salvation is near"),
        OracleEmbeddingOptions.builder()
            .model("ALL_MINILM_L12_V2")
            .batching(false)
            .metadataMode(MetadataMode.ALL)
            .preferences(OracleEmbeddingPreferences.builder()
                .provider("database")
                .model("ALL_MINILM_L12_V2")
                .build())
            .build()));
----

== Sample Controller

[source,java]
----
@RestController
public class EmbeddingController {

    private final EmbeddingModel embeddingModel;

    public EmbeddingController(EmbeddingModel embeddingModel) {
        this.embeddingModel = embeddingModel;
    }

    @GetMapping("/ai/embedding")
    public Map<String, Object> embed(@RequestParam(defaultValue = "Tell me a joke") String message) {
        EmbeddingResponse embeddingResponse = this.embeddingModel.embedForResponse(List.of(message));
        return Map.of("embedding", embeddingResponse);
    }
}
----

== Manual Configuration

If you are not using Spring Boot, you can manually configure `OracleEmbeddingModel`.
For this add the `spring-ai-oracle` dependency to your project's Maven `pom.xml` file:

[source,xml]
----
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-oracle</artifactId>
</dependency>
----

or to your Gradle `build.gradle` build file.

[source,groovy]
----
dependencies {
    implementation 'org.springframework.ai:spring-ai-oracle'
}
----

TIP: Refer to the xref:getting-started.adoc#dependency-management[Dependency Management] section to add the Spring AI BOM to your build file.

[source,java]
----
@Bean
EmbeddingModel embeddingModel(DataSource dataSource) {
    return OracleEmbeddingModel.builder(dataSource)
        .defaultOptions(OracleEmbeddingOptions.builder()
            .model("ALL_MINILM_L12_V2")
            .preferences(OracleEmbeddingPreferences.builder()
                .provider("database")
                .model("ALL_MINILM_L12_V2")
                .build())
            .build())
        .build();
}
----
